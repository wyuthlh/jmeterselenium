Selenium

Functional Testing with Selenium

Getting Started

  1. Go to this website jmeter-plugins.org and click “Download”.

  2. Selenium/Web Driver - copy paste all the lib files from the downloads to apache jmeter lib folder.

  3. Go to this Index of /114.0.5735.90/ website and click “chromedriver_win32.zip”.

  4. Chromedriver - Download the exe and paste the path under the licenses.

  5. Restart Jmeter > Add Thread Group.

  6. Under Thread Group > Add Sampler > WebDriver Sampler > Add Config Element > Chrome Driver Config.

  7. Place the location of chromedriver inside the licenses folder.

  8. Add > Listener > View Result Tree.

  9. Add > Listener > View Result in Table.

  10. Save the file in Jmeter.

  11. Can also add the user, seconds and also loop count to view the performance testing.

  12. For using Selenium in JMeter, the tester can write the codes here at ‘WebDriver Sampler’.

Sidebar Navigation and Settings Access on Carebiz

Description:

  1. Open Sidebar: Clicks the sidebar button and checks if the sidebar opens.

  2. Access Settings: Selects the settings option from the sidebar and verifies the settings menu opens.

  3. Navigate to Account Settings: Clicks the account button to view account-related options.

  4. Select and View General Settings: Selects “General” and verifies that the page loads.

  5. Return to Previous Menu: Uses the back button to navigate to the main settings menu.

  6. Close Settings: Clicks to close the settings and verifies it’s closed

Steps:

  1. Open Sidebar: Click the three-line menu button to reveal the sidebar.

  2. Access Settings: Select the “Settings” option from the sidebar.

  3. Navigate to Account Settings: Click on the “Account” option within the settings menu.

  4. Select and View General Settings: Click on “General” under the account settings to view general account information.

  5. Return to Previous Menu: Use the “Back” button to navigate back to the main settings menu.

  6. Close Settings: Click the close button to exit the settings menu and return to the main page.

Expected Results:

  1. Open Sidebar: The sidebar should open and display all available menu options when the three-line button is clicked.

  2. Access Settings: The settings menu should load, showing various settings options like Account, Privacy, etc.

  3. Navigate to Account Settings: The user should be able to view account-related settings, confirming the navigation is successful.

  4. Select and View General Settings: The “General” settings page should load and display relevant general account details.

  5. Return to Previous Menu: The main settings menu should reappear, indicating the user has navigated back successfully.

  6. Close Settings: The settings menu should close, bringing the user back to the main application screen or the previous active page.

Sidebar Settings and Logout Functionality on Carebiz

Description: This test validates the Settings and Logout functionalities via the sidebar menu. It ensures that users can open the sidebar, access the settings menu, navigate to the “About” section to view the debug log, close the settings, and then successfully log out. This sequence is essential for testing both the application’s settings access and user session management.

Steps:

  1. Open Sidebar: Click the three-line button to open the sidebar.

  2. Access Settings: Select the “Settings” option from the sidebar menu.

  3. Navigate to About Section: Click on the “About” option within the settings menu.

  4. View Debug Log: Click the “Show Debug Log” button to display the debug log.

  5. Close Debug Log Tab: Close the debug log tab and return to the main screen.

  6. Open Sidebar Again: Click the three-line button to reopen the sidebar.

  7. Log Out: Select the “Logout” option from the sidebar menu to log out of the application.

Expected Result:

  1. Open Sidebar: The sidebar should open, displaying all available options.

  2. Access Settings: The settings menu should appear, showing various options like Account, Privacy, About, etc.

  3. Navigate to About Section: The user should be able to access the “About” section, confirming that the navigation is successful.

  4. View Debug Log: The debug log should be displayed, showing technical details or logs as expected.

  5. Close Debug Log Tab: The debug log should close, and the user should return to the main settings or the previous screen.

  6. Open Sidebar Again: The sidebar should reopen, displaying menu options including “Logout.”

  7. Log Out: The user should be logged out of the application and redirected to the login page or the application’s main landing page.
